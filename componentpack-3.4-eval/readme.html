Introduction

JComponentPack is a Java Swing GUI library, a visual JavaBeans collection, it's based on the Swing MVC architecture and 100% pure Java, it includes more than 20 components that Swing haven't, such as excel style Java spreadsheet component, database enabled Java table component, calendar, list view, wizard framework and many more, these beans can drag to your GUI form directly, all the components are ready to use, they can improve your productivity, accelerate your project development process, save time, reduce costs for your team.

System Requirements

Java Runtime Environment 1.5.0 or later.

Contents of the JComponentPack

This section contains a general summary of the files and directories in the JComponentPack.
Libraries
(In the lib/ subdirectory) The JComponentPack standard libraries and optional libraries.
Plug-ins
(In the plugins/ subdirectory) The JComponentPack Plug-in for NetBeans IDE, also contains the merge properties file plug-in for free to use.
Web Components
(In the web/ subdirectory) The web components and demonstration of the JComponentPack product.
Documentation - Requires JComponentPack Binary License, Order Now!
(In the docs/ subdirectory) The Programmer Guide and Javadoc API of the JComponentPack product.
Demo Source Code - Requires JComponentPack Binary License, Order Now!
(In the demosrc/ subdirectory) The demo source code of the JComponentPack product.
Source Code - Requires JComponentPack Source Code License, Order Now!
(In the src/ subdirectory) The source code of the JComponentPack swing components.
Web Components Source Code - Requires JComponentPack Source Code License, Order Now!
(In the websrc/ subdirectory) The source code of the JComponentPack web components.
Notes: The above contents mark "green" is both available in JComponentPack Binary License or JComponentPack Source Code License, the above contents mark "red" is only available in JComponentPack Source Code License.

Redistribution

The componentpack.jar and license.jar is the only required file you need to distribute with your desktop applications or Applet, the other package is optional, they can be deployed lazily on the Applet environment, the core library of JComponentPack swing components is very small after compression with pack200.

The componentpack-web.jar and license.jar is the only required file you need to distribute with your web applications.
